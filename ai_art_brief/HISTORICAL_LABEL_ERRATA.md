# 历史标签校正与禁用清单

这个文件按项目 skills 的规则写：它只处理美术样张中的历史标签边界，不把样张当史实权威层。

## 适用对象

- `images/00_primary_pressure_network_kanyu_reference.png`
- `images/01_style_target_pressure_network_concept.png`

这两张图都是美术方向参考。它们可以说明画面气质、符号层级、压力路网和堪舆总图方向，但不能直接证明地名、关隘、路线、年份或边界。

## 已校正标签

| 原标签 | 当前处理 | 说明 |
| --- | --- | --- |
| 山海关 | 改为 `榆关（待校）` | `山海关` 不应作为北宋标签使用。若东北/辽西走廊需要关口，只能先用候选名，并标 `待校`。 |
| 居庸关 | 标为 `居庸关（辽境）` | 关隘地理位置可作为燕云/辽境压力源出现，但不能画成北宋朝廷直接控制的宋境关口。 |

## 硬性禁用

- 北宋全国图不要出现 `山海关`。
- 不要把辽境、夏境、大理、吐蕃诸部的节点画成宋境行政节点。
- 不要把候选关口、候选驿路、候选水道写成已考定史实。
- 不要把谭图线稿、CHGIS 外推面、程序补面直接说成最终边界。
- 不要使用现代省界、国界、高速、铁路、机场、现代城市名。

## 关隘标签规则

每个关隘/山口/边防节点进入最终图前，至少要带这些状态：

| 字段 | 允许值示例 |
| --- | --- |
| `period_status` | `song_official`、`liao_yanyun`、`western_xia`、`candidate`、`anachronism_replaced` |
| `political_context` | `宋境`、`辽境`、`夏境`、`边地候选`、`待校` |
| `confidence` | `high`、`medium`、`low`、`needs_source_check` |
| `source_role` | `CHGIS`、`谭图参考`、`地方志待校`、`玩法候选` |

## 给出图模型的限制语句

可以直接复制给 Gemini / GPT / Kimi：

> 这是一张北宋压力网络堪舆图，不得出现明代以后才成立的关隘名。禁止把 `山海关` 写入北宋图。`居庸关` 如出现，只能标作 `居庸关（辽境）` 或 `燕云压力源`，不能画成宋境节点。所有不确定关口、驿站、水道必须标 `待校`，不得冒充史实定稿。

## 后续校准顺序

1. 先用项目行政点、路网、水系和已有谭图参考确认位置关系。
2. 再按 CHGIS / 谭图 / 地方志 / 专门史料补证地名。
3. 最终进入 Unity 或美术底图前，生成一份标签审查清单，专门查时代错置、政权归属错置和现代名混入。
